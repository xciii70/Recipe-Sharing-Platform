## 1. Introduction
**Kitchenary** is a highly responsive web platform tailored for food enthusiasts and developers alike. It bridges data-driven backend architectures with clean interface design, allowing users to register accounts, explore culinary ideas, document creative recipes, and review active system team member profiles seamlessly.

## 2. Getting Started & Authentication

Before accessing the recipe collection, all users must log in securely via the centralized authentication portal.

### Sign Up / Registration
* If you do not have an active account, navigate to the landing page and click on the **Sign Up** hyperlink below the credential card.
* Enter a valid username and safe password credentials.
* Click the **Sign Up** submission action button to initialize your database user record.

### Log In
1. Provide your standard username/email and password on the compact authentication layout sheet.
2. Click the block-level **Login** button. 
3. Upon successful validation, the system will instantly redirect you to the main dashboard.

### Logging Out
* To terminate your active session securely, click the pill-shaped **Logout** button situated at the top-right corner of the persistent header navigation bar.

## 3. Navigating the Dashboard

The global top navigation bar (`navbar`) is sticky and remains locked to the top of your viewport as you scroll through content layers. Use it to jump directly between core system pages:
* **Home Page**: Returns you instantly to your dynamic dashboard stream.
* **Group Members**: Renders the complete profile cards of the core technical team.

## 4. Managing Recipes

The homepage loads a responsive grid that automatically dynamically scales food preview cards based on your physical screen width boundaries.

### Viewing Recipes
* Each recipe tile exhibits a clean preview framework consisting of a dedicated food cover image (or a minimal fallback placeholder icon if unassigned) along with the **Recipe Title** and the explicit **Creator's name** (e.g., *Salad by Julia* or *Steak by Tim*).

### Creating a New Recipe
1. Click the absolute floating circular **`+` (Plus)** action button fixed at the lower-right corner of your screen.
2. Complete the centralized structured recipe configuration panel:
   * **Recipe Title**: Provide a descriptive, memorable name.
   * **Ingredients / Directions**: Supply the precise structural steps in the provided multiline textarea field (the scroll mechanism automatically constraints inputs cleanly).
   * **Cover Image**: Upload a valid media file representation.
3. Click the block-level submission button to commit your record directly to the data layer.

### Modifying or Deleting a Recipe
* While reviewing standalone detailed read-only showcase sheets belonging to your created recipes, inline interactive modification anchors are provided at the base:
  * **Edit Recipe**: Modifies stored structural text string values without breaking data layouts.
  * **Delete Recipe**: Permanently eliminates the selected formula record safely from database instances with high contrast warning colors.

## 5. Viewing Group Members

By navigating to the **Group Members** sector via the navbar, you can interact with the dynamic team identity badges:
* **Structured Alignment Lines**: Information fields are hard-aligned at `100px` labels, forcing colons and dynamic credentials to lock vertically for maximum readability.
* **Embedded Bios**: The concave inset custom profile area respects precise user text-line breaks and handles continuous literal string feeds smoothly, preventing unwanted UI leaks.
